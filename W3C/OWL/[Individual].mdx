```turtle
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix ex: <http://example.org/> .

{Individual.uri} a owl:NamedIndividual ;
    rdfs:label "{Individual.label}"@en ;
    rdfs:comment "{Individual.comment}"@en .

# Type assertions
{Individual.types && Individual.types.map(type => `{Individual.uri} a ${type} .`).join('\n')}
```

### Identity Assertions

```turtle
# Same individual, different URIs
{Individual.uri} owl:sameAs ex:alternativeURI .

# Different individuals
{Individual.uri} owl:differentFrom ex:otherIndividual .

# All different
ex:individual1 a owl:NamedIndividual .
ex:individual2 a owl:NamedIndividual .
ex:individual3 a owl:NamedIndividual .

[ a owl:AllDifferent ;
  owl:distinctMembers (ex:individual1 ex:individual2 ex:individual3)
] .
```

### Negative Property Assertions

```turtle
# Stating what is NOT true
{Individual.uri} a owl:NamedIndividual .

[] a owl:NegativePropertyAssertion ;
   owl:sourceIndividual {Individual.uri} ;
   owl:assertionProperty ex:hasProperty ;
   owl:targetIndividual ex:target .
# Explicitly states: {Individual.uri} does NOT have ex:hasProperty ex:target
```

### SPARQL Queries

```sparql
PREFIX owl: <http://www.w3.org/2002/07/owl#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

DESCRIBE {Individual.uri}

# Find all properties and values
SELECT ?property ?value
WHERE {
  {Individual.uri} ?property ?value .
}

# Find all types
SELECT ?type
WHERE {
  {Individual.uri} a ?type .
  FILTER(?type != owl:NamedIndividual)
}

# Find individuals of the same type
SELECT ?other ?label
WHERE {
  {Individual.uri} a ?type .
  ?other a ?type .
  FILTER(?other != {Individual.uri})
  OPTIONAL { ?other rdfs:label ?label }
}

# Find equivalent individuals (via sameAs)
SELECT ?equivalent
WHERE {
  {Individual.uri} owl:sameAs ?equivalent .
}
UNION
WHERE {
  ?equivalent owl:sameAs {Individual.uri} .
}
```

## OWL Reasoning

### Inferences

Reasoners will infer the following about this individual:

1. **Type Inference**: Inferred types based on property domain/range
2. **Equality**: Merging of facts about equivalent individuals (owl:sameAs)
3. **Property Values**: Inferred property values via transitivity, symmetry, etc.
4. **Class Membership**: Membership in superclasses

## Named vs Anonymous Individuals

This is a **named individual** with a persistent URI identifier.

### Named Individual

- Has a URI that can be referenced
- Suitable for important, reusable entities
- Can be shared across datasets
- Example: `{Individual.uri}`

### Anonymous Individual (Blank Node)

```turtle
# For comparison, anonymous individuals:
ex:person ex:address [
  a ex:Address ;
  ex:street "123 Main St" ;
  ex:city "Springfield"
] .
```

## Best Practices

When working with `{Individual.label}`:

1. **Unique Names Assumption (UNA)**: OWL doesn't assume UNA; use owl:sameAs or owl:differentFrom explicitly
2. **Identity**: Use owl:sameAs to link equivalent URIs across datasets
3. **Distinction**: Use owl:differentFrom when entities might be confused
4. **URIs**: Choose persistent, dereferenceable URIs
5. **Documentation**: Provide rdfs:label and rdfs:comment
6. **Linked Data**: Link to related individuals in other datasets
7. **Validation**: Verify compliance with class restrictions

## Common Patterns

### Person Individual

```turtle
ex:johnDoe a owl:NamedIndividual ,
             ex:Person ;
    ex:firstName "John" ;
    ex:lastName "Doe" ;
    ex:email "john.doe@example.org" ;
    ex:birthDate "1980-05-15"^^xsd:date ;
    ex:knows ex:janeDoe ;
    owl:sameAs <http://dbpedia.org/resource/John_Doe> .
```

### Organization Individual

```tuple
ex:acmeCorp a owl:NamedIndividual ,
              ex:Organization ;
    ex:legalName "Acme Corporation" ;
    ex:founded "1950"^^xsd:gYear ;
    ex:hasEmployee ex:johnDoe ;
    ex:homepage <https://www.acme.example.org> .
```

### Location Individual

```turtle
ex:springfield a owl:NamedIndividual ,
                 ex:City ;
    ex:cityName "Springfield" ;
    ex:state ex:illinois ;
    ex:country ex:usa ;
    ex:population 116250 ;
    geo:lat "39.7817"^^xsd:decimal ;
    geo:long "-89.6501"^^xsd:decimal .
```

## Validation

### Class Membership Validation

Ensure the individual satisfies all restrictions of its classes:

```turtle
ex:PersonWithParents a owl:Class ;
    rdfs:subClassOf ex:Person ;
    rdfs:subClassOf [
        a owl:Restriction ;
        owl:onProperty ex:hasParent ;
        owl:cardinality 2
    ] .

# Valid
ex:john a ex:PersonWithParents ;
    ex:hasParent ex:parent1 , ex:parent2 .

# Invalid - would be caught by reasoner
# ex:jane a ex:PersonWithParents ;
#     ex:hasParent ex:parent1 .  # Only one parent
```
