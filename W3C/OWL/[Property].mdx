## OWL Reasoning

### Inference Rules

Reasoners apply the following inferences:

1. **Domain Inference**: If `x {Property.uri} y`, then `x` is of type `domain`
2. **Range Inference**: If `x {Property.uri} y`, then `y` is of type `range`
3. **Sub-Property Inference**: If `P ⊆ {Property.uri}` and `x P y`, then `x {Property.uri} y`
4. **Inverse Inference**: If defined, `x {Property.uri} y` implies `y {Property.inverseOf} x`

## Property Restrictions

### Universal Restriction (allValuesFrom)

```turtle
ex:RestrictedClass a owl:Class ;
    rdfs:subClassOf [
        a owl:Restriction ;
        owl:onProperty {Property.uri} ;
        owl:allValuesFrom ex:TargetClass
    ] .
# All values of this property must be of TargetClass
```

### Existential Restriction (someValuesFrom)

```turtle
ex:SomeValuesClass a owl:Class ;
    rdfs:subClassOf [
        a owl:Restriction ;
        owl:onProperty {Property.uri} ;
        owl:someValuesFrom ex:TargetClass
    ] .
# At least one value must be of TargetClass
```

### Cardinality Restrictions

```turtle
ex:CardinalityClass a owl:Class ;
    rdfs:subClassOf [
        a owl:Restriction ;
        owl:onProperty {Property.uri} ;
        owl:minCardinality 1 ;
```

```turtle
ex:QualifiedClass a owl:Class ;
```

## Best Practices

When using `{Property.label}`:

1. **Type Safety**: Ensure subjects and objects match domain and range
2. **Characteristics**: Declare functional, symmetric, transitive properties appropriately
3. **Inverse Properties**: Define inverses to enable bidirectional queries
4. **Documentation**: Provide comprehensive labels, comments, and examples
5. **Validation**: Use reasoners to check consistency
6. **Performance**: Consider materialization vs. on-demand reasoning
7. **Modularity**: Group related properties in coherent modules
