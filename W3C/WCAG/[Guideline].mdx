## Purpose and Benefits

### Who Benefits

This guideline helps users who:

- Use assistive technologies (screen readers, screen magnifiers, speech recognition)
- Have visual, auditory, motor, or cognitive disabilities
- Use alternative input methods or navigation techniques
- Access content in challenging environments or on diverse devices
- Experience temporary disabilities or situational limitations

### Impact on User Experience

Implementing this guideline improves:

- **Accessibility**: Content is usable by people with disabilities
- **Usability**: Better experience for all users, not just those with disabilities
- **Compatibility**: Works across different technologies and platforms
- **Flexibility**: Adapts to user preferences and needs
- **Robustness**: Maintains functionality across various contexts

## Common Techniques

### Sufficient Techniques

These techniques are reliable ways to meet the success criteria:

1. **Semantic HTML**: Use proper HTML elements for their intended purpose
2. **ARIA Attributes**: Apply ARIA when HTML semantics are insufficient
3. **Alternative Text**: Provide text alternatives for non-text content
4. **Keyboard Support**: Ensure all functionality is keyboard accessible
5. **Focus Management**: Maintain logical and visible focus order
6. **Error Identification**: Clearly identify and describe errors
7. **Labels and Instructions**: Provide clear labels for form inputs

### Advisory Techniques

These techniques go beyond requirements but improve accessibility:

- Progressive enhancement strategies
- Clear and simple language
- Consistent navigation and identification
- Helpful error suggestions
- Multiple ways to find content
- Sufficient time to read and interact

### Common Failures

Avoid these practices that violate this guideline:

- Missing or incorrect semantic structure
- Keyboard traps or inaccessible controls
- Insufficient color contrast
- Missing alternative text
- Automatic redirects or updates without user control
- Time limits without options to extend
- Content that causes seizures or physical reactions

## Conformance Considerations

### Level A (Minimum)

Essential requirements that, if not met, make content inaccessible to some users. All websites should meet Level A criteria.

### Level AA (Recommended)

Addresses major barriers to accessibility. Level AA is the recommended target for most organizations and is required by many laws and policies.

### Level AAA (Enhanced)

Provides the highest level of accessibility. Level AAA may not be achievable for all content but should be applied where possible.
