### Screen Reader Testing

#### NVDA (Windows)

1. Start NVDA (Insert + N)
2. Navigate to the element (Arrow keys or Tab)
3. Verify announced information is correct
4. Test all interactive elements (Enter/Space)
5. Check that relationships are clear

#### VoiceOver (Mac)

1. Enable VoiceOver (Cmd + F5)
2. Navigate with VO keys (Control + Option + Arrow)
3. Verify rotor categories (VO + U)
4. Test interactions (VO + Space)
5. Check announcements are appropriate

#### JAWS (Windows)

1. Start JAWS
2. Use virtual cursor (Arrow keys)
3. Test forms mode if applicable (Enter on form field)
4. Verify element list (Insert + F3)
5. Check heading navigation (H key)

## Common Issues and Solutions

### Issue 1: Incomplete Implementation

**Problem**: Technique partially implemented, missing key aspects.

**Solution**:
```javascript
// Incorrect - Missing required attributes
<div>
  <button>Click me</button>
</div>

// Correct - Full implementation of {technique.id}
<div role="region" aria-label="Interactive controls">
  <button
    type="button"
    aria-label="Descriptive action label"
  >
    Click me
  </button>
</div>
```

### Issue 2: Browser Compatibility

**Problem**: Technique not supported in older browsers.

**Solution**: Provide progressive enhancement or polyfill:

```javascript
// Feature detection and fallback
if ('attribute' in element) {
  // Modern implementation
  element.setAttribute('aria-label', 'Modern approach')
} else {
  // Fallback for older browsers
  element.title = 'Fallback approach'
}
```

### Issue 3: Dynamic Content

**Problem**: Technique doesn't work with dynamically added content.

**Solution**:
```javascript
// Use MutationObserver to handle dynamic content
const observer = new MutationObserver((mutations) => {
  mutations.forEach((mutation) => {
    if (mutation.type === 'childList') {
      // Reapply technique {technique.id} to new elements
      applyTechnique(mutation.addedNodes)
    }
  })
})

observer.observe(document.body, {
  childList: true,
  subtree: true
})
```
