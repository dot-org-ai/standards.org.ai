## Description

<Description>
{context.description}

The {context.name} context ({context.uri}) provides JSON-LD bindings for the {context.namespace} vocabulary. This context enables developers to use simple property names in JSON-LD documents that automatically expand to full IRIs from the {context.name} namespace. It is maintained by {context.maintainer} and follows the JSON-LD 1.1 specification.
</Description>

### Using jsonld.js

<CodeExample language="javascript" title="Processing with jsonld.js">
{`import jsonld from 'jsonld';

const document = {
  "@context": "${context.uri}",
  "@type": "${context.namespace.includes('schema.org') ? 'Person' : 'Resource'}",
  "name": "Example"
};

// Expand to full IRIs
const expanded = await jsonld.expand(document);
console.log('Expanded:', JSON.stringify(expanded, null, 2));

// Output:
// [
//   {
//     "@type": ["${context.namespace}${context.namespace.includes('schema.org') ? 'Person' : 'Resource'}"],
//     "${context.namespace}name": [
//       { "@value": "Example" }
//     ]
//   }
// ]

// Compact with different context
const compacted = await jsonld.compact(expanded, {
  "@context": "${context.uri}"
});
console.log('Compacted:', JSON.stringify(compacted, null, 2));

// Convert to N-Quads
const nquads = await jsonld.toRDF(document, {
  format: 'application/n-quads'
});
console.log('N-Quads:', nquads);

// Normalize for signing/comparison
const normalized = await jsonld.normalize(document, {
  algorithm: 'URDNA2015',
  format: 'application/n-quads'
});
console.log('Normalized:', normalized);`}
</CodeExample>

## Best Practices

### Context Management

1. **Use Stable URLs**: Always reference stable, versioned context URLs
   ```json
   {
     "@context": "${context.uri}"
   }
   ```

2. **Cache Contexts**: Implement aggressive caching for remote contexts
   ```javascript
   const documentLoader = jsonld.documentLoaders.node();
   const customLoader = async (url) => {
     if (url === '${context.uri}') {
       return {
         contextUrl: null,
         document: cachedContext,
         documentUrl: url
       };
     }
     return documentLoader(url);
   };
   ```

3. **Combine Contexts**: Merge standard and custom contexts
   ```json
   {
     "@context": [
       "${context.uri}",
       {
         "custom": "https://example.org/vocab/"
       }
     ]
   }
   ```

4. **Version Awareness**: Be aware of context versions
   ${context.version ? `- Current version: ${context.version}` : ''}
   - Monitor for updates and deprecations
   - Test compatibility before upgrading

### Performance Optimization

1. **Inline Critical Contexts**: For high-performance scenarios, inline frequently-used contexts
2. **HTTP/2**: Use HTTP/2 for parallel context loading
3. **CDN**: Host contexts on CDN for global distribution
4. **Compression**: Enable gzip/brotli compression for context files
5. **Preload**: Use `<link rel="preload">` for context URLs

### Error Handling

<CodeExample language="javascript" title="Robust Error Handling">
{`async function processJsonLd(document) {
  try {
    const expanded = await jsonld.expand(document, {
      documentLoader: customLoader,
      safe: true, // Prevent remote context loading in production
      timeout: 5000 // 5 second timeout
    });
    return expanded;
  } catch (error) {
    if (error.name === 'jsonld.LoadDocumentError') {
      console.error('Failed to load context:', error.message);
      // Fallback to cached context
    } else if (error.name === 'jsonld.InvalidFrame') {
      console.error('Invalid JSON-LD frame:', error.message);
    } else {
      console.error('JSON-LD processing error:', error);
    }
    throw error;
  }
}`}
