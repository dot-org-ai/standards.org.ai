{unit.code === 'px' && `
### Pixel Density

- 1px = 1/96th of 1 inch (CSS reference pixel)
- On high-DPI displays, 1 CSS pixel may equal multiple device pixels
- Device Pixel Ratio (DPR) affects how px units render
- Modern displays often have DPR of 2x or 3x
`}

{unit.code === 'em' && `
### Calculation Examples

\`\`\`
Parent font-size: 16px
1em = 16px
2em = 32px
0.5em = 8px

Nested element with parent font-size: 20px
1em = 20px (relative to immediate parent)
\`\`\`

### Compounding Effect

em units compound when nested:
- Parent: 1.2em of 16px = 19.2px
- Child: 1.2em of 19.2px = 23.04px
- Grandchild: 1.2em of 23.04px = 27.65px
`}

{unit.code === 'rem' && `
### Calculation Examples

\`\`\`
Root font-size: 16px (default)
1rem = 16px
2rem = 32px
0.5rem = 8px

Root font-size: 20px (custom)
1rem = 20px
1.5rem = 30px
\`\`\`

### No Compounding

Unlike em, rem always refers to root font-size:
- All levels: 1rem = same value throughout document
- Predictable and easier to maintain
- Preferred for spacing systems
`}

{unit.code === 'vw' && `
### Viewport Width Calculation

\`\`\`
Viewport width: 1200px
1vw = 12px
50vw = 600px (half viewport)
100vw = 1200px (full viewport)

Viewport width: 375px (mobile)
1vw = 3.75px
50vw = 187.5px
\`\`\`
`}

{unit.code === 'vh' && `
### Viewport Height Calculation

\`\`\`
Viewport height: 800px
1vh = 8px
50vh = 400px (half viewport)
100vh = 800px (full viewport)

Mobile viewport (without URL bar): 667px
1vh = 6.67px
100vh = 667px
\`\`\`

### Mobile Browser Considerations

Mobile browsers may change viewport height when UI elements (address bar, toolbars) show/hide. Consider using dvh (dynamic viewport height) for more stable behavior.
`}

{unit.code === '%' && `
### Percentage Calculation Context

Percentages resolve differently based on the property:

\`\`\`
width: 50%        → 50% of parent's width
height: 50%       → 50% of parent's height
font-size: 150%   → 150% of parent's font-size
line-height: 120% → 120% of element's font-size
padding: 10%      → 10% of parent's WIDTH (yes, even for padding-top/bottom)
margin: 5%        → 5% of parent's WIDTH
transform: translate(50%, 50%) → 50% of element's own width/height
\`\`\`
`}

{unit.type === 'relative' ? `

{unit.code === 'px' && `
- Avoid using px for font-size (limits user zoom)
- Good for: borders (1px), small icons, media query breakpoints
- Use rem or em for most spacing and typography
- Remember that CSS px ≠ device pixels on high-DPI screens
`}

{unit.code === 'em' && `
- Be aware of compounding in nested elements
- Use for component-specific spacing
- Good for padding/margin related to local font-size
- Can make components harder to predict in deep nesting
- Consider rem for non-component-scoped values
`}

{unit.code === 'rem' && `
- Excellent for global spacing systems (0.5rem, 1rem, 2rem)

- Use for consistent typography scale
- Respects user font-size preferences
- Easier to maintain than em in complex layouts

{unit.code === 'vw' || unit.code === 'vh' ? `

- Test on actual mobile devices, not just browser DevTools

{unit.code === '%' && `

- Good for: max-width, opacity, transform translate

## Performance Considerations

{unit.type === 'relative' ? `
- Relative units require calculation at render time
- Can trigger reflows if reference values change
- Generally negligible impact on modern browsers
- Use CSS custom properties to avoid repeated calculations
` : `
- Absolute units are computed once
- Simpler for browsers to calculate
- May perform slightly better in complex layouts
- Differences are typically imperceptible
`}

{unit.code === 'vw' || unit.code === 'vh' ? `
- Viewport units recalculate on resize
- Can cause performance issues if overused
- Be cautious with vw/vh in animations
- Use CSS containment to limit reflow scope
`}
