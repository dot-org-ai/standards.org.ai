## Verifiable Data Registry

**Registry Type:** {Method.features.verifiableDataRegistry}

{Method.name === 'web' && (
  <>
    The `did:web` method uses **web servers** as the verifiable data registry. DID documents
    are hosted at predictable HTTPS URLs derived from the DID identifier.

    **Resolution URL Pattern:**
    <CodeExample language="text">
{`did:web:example.com          → https://example.com/.well-known/did.json
did:web:example.com:user:alice → https://example.com/user/alice/did.json`}
    </CodeExample>
  </>
)}

{Method.name === 'key' && (
  <>
    The `did:key` method uses **cryptographic key material** as the verifiable data registry.
    No external storage is required; the DID document is derived from the key itself.
  </>
)}

{Method.name === 'ion' && (
  <>
    The `did:ion` method uses the **Bitcoin blockchain** and **IPFS** as the verifiable data
    registry. Operations are anchored to Bitcoin for ordering and timestamping, while
    actual data is stored on IPFS.
  </>
)}

{Method.name === 'ethr' && (
  <>
    The `did:ethr` method uses the **Ethereum blockchain** as the verifiable data registry.
    DID documents are derived from Ethereum addresses, with updates recorded via smart contracts.
  </>
)}

{Method.name === 'peer' && (
  <>
    The `did:peer` method uses **peer-to-peer exchanges** as the verifiable data registry.
    No blockchain or centralized registry is required; DIDs are exchanged directly between parties.
  </>
)}
