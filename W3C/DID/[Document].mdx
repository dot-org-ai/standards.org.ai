## Validation Rules

### Property-Specific Validation

{Document.property === 'id' && (
  <>
    1. MUST be a valid DID conforming to the DID syntax specification
    2. MUST match the DID that was resolved
    3. MUST NOT contain fragment identifiers
    4. MUST be a string value
  </>
)}

{Document.property === 'verificationMethod' && (
  <>
    1. Each verification method MUST have an 'id' property
    2. Each verification method MUST have a 'type' property
    3. Each verification method MUST have a 'controller' property
    4. Each verification method MUST include exactly one verification material property (publicKeyJwk, publicKeyMultibase, etc.)
    5. The 'id' property MUST be a DID URL that can be dereferenced
    6. The 'controller' property MUST be a valid DID
  </>
)}

{Document.property === 'authentication' && (
  <>
    1. Can contain either verification method references (strings) or embedded verification methods (objects)
    2. Referenced verification methods MUST exist in the 'verificationMethod' array or be resolvable
    3. Embedded verification methods MUST conform to verification method requirements
    4. MUST be used to authenticate the DID subject
  </>
)}

{Document.property === 'service' && (
  <>
    1. Each service MUST have an 'id' property
    2. Each service MUST have a 'type' property
    3. Each service MUST have a 'serviceEndpoint' property
    4. The 'id' property MUST be a DID URL
    5. The 'serviceEndpoint' can be a URI string, map, or set
  </>
)}

### General Validation

<CodeExample language="javascript">
{`function validateDIDDocument(doc) {
  // Check required properties
  if (!doc.id) {
    throw new Error('DID document missing required "id" property');
  }

  // Validate DID syntax
  if (!isValidDID(doc.id)) {
    throw new Error('Invalid DID format in "id" property');
  }

  // Validate @context
  if (!doc['@context']) {
    throw new Error('DID document missing @context');
  }

  // Ensure @context includes DID v1 context
  const contexts = Array.isArray(doc['@context'])
    ? doc['@context']
    : [doc['@context']];

  if (!contexts.includes('https://www.w3.org/ns/did/v1')) {
    throw new Error('DID document @context must include https://www.w3.org/ns/did/v1');
  }

  // Validate verification methods
  if (doc.verificationMethod) {
    doc.verificationMethod.forEach(vm => {
      if (!vm.id || !vm.type || !vm.controller) {
        throw new Error('Invalid verification method structure');
      }
    });
  }

  return true;
}`}
</CodeExample>

When resolving a DID document, this property is processed as follows:

1. Parse the property value according to its data type
2. Validate the property structure
3. Verify all references (for verification relationships)
4. Resolve any embedded contexts
5. Return the processed property value

## Security Considerations

{Document.property === 'verificationMethod' && (

    ### Key Management
    - Protect private keys corresponding to public keys in verification methods
    - Implement key rotation policies
    - Use hardware security modules for high-value keys
    - Never include private key material in DID documents

    - Ensure verification methods use strong cryptographic algorithms
    - Validate that verification method controllers are authorized

    - Never use the same key for authentication and other purposes without careful consideration

<CodeExample language="javascript">
{`// Update a DID document property
async function updateDIDDocument(did, updates) {
  // Retrieve current document
  const current = await resolveDID(did);

  // Apply updates
  const updated = {
    ...current.didDocument,
    ...updates
  };

  // Validate updated document
  validateDIDDocument(updated);

  // Publish update
  await publishDIDDocument(did, updated);

  return updated;
}`}
</CodeExample>

### Best Practices

1. **Minimal Disclosure**: Only include necessary information in DID documents
2. **Key Hygiene**: Use separate keys for different verification relationships
3. **Regular Updates**: Keep DID documents current with active keys and services
4. **Validation**: Always validate documents before use
5. **Privacy**: Consider correlation risks when using public DID documents
